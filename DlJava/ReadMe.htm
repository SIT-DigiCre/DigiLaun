このソフトについて

Digi Laun というのは、芝浦祭等に出展する作品向けのランチャープログラムです。

目次へ

要求スペック

  • OS : 任意
  • ソフト : JRE 6 以降 (ダウンロード)

使い方

バージョン 3.0 からの新機能!!

Digi Laun を頒布用と展示用で使い分けられるようになりました。 config.jar でモードを「展示モード」「頒布モード」から選択できます。頒布モードには以下の特徴があります。
  • 時間無制限で、ウィンドウ上の帯がずっと緑色のままです
  • 隠しキーコマンドが使えません
  • 普通にウィンドウで表示します (展示モードでは全画面になります)
  • [閉じる] ボタンで終了できます

DigiLaun.jar を実行すると以下のようなウィンドウが出てきます。

作品のボタンを押すと概要ウィンドウが出てきます。

「もどる」ボタンで概要ウィンドウを閉じます。
「はじめる！」ボタンを押すとプログラムを起動します。
この時「もどる」ボタンが変化して別のボタンになります。

これを押すとプログラムを強制終了します。緊急時にお使いください。

また、しばらく起動しているとウィンドウ上部が徐々に変化していきます。時間の目安にしましょう。

コンフィグについて

作品の登録方法

　まず必要なものを用意してください。

  • 作品本体
    • exe以外も開けます
  • 作品のタイトルや簡単な遊び方などを記した概要イメージ
    • サイズ無制限
      • ドット絵をきれいに表示させたいなら横640px、縦480px
  • ボタンに貼り付けるアイコン画像
    • これも実質サイズ無制限、でも横長が基本
      • 縦横比は 横2:縦1 くらいが適当
      • ドット絵をきれいに表示させたいなら横128pxくらい、縦64px
  • 著作権その他の情報をまとめたテキストファイル
    • txt または RTF, HTML 形式
      • フリー素材に頼ってる場合はライセンスをきちんと記そう
      • txt はシフト JIS のみ対応
      • RTF は現在、日本語に対応してないみたいです。うわさでは文字コードを UTF-8 にするといいとか…
    • 実は PDF などでも可 (コンフィグにてファイル選択時に「ファイルの種類」をクリックすると…)

　画像として使用可能な形式は、GIF, JPEG, PNG (αチャンネル対応) です。

　次にこれらを適当なフォルダにまとめます。
　そしたら DigiLaun.jar があるフォルダに「config.jar」というファイルがあるので、これを実行してください。

表の編集箇所をダブルクリックすると編集できます。以下、列の説明です。

「入力デバイス」
アプリの入力に使うデバイス（マウス、キーボード、ゲームパッドなど）です。アプリでなければ空欄で大丈夫です。
「パス」
例えばゲームソフトならEXE、CGならPNG、DTM作品ならMP3といったふうに、作品を見るときにダブルクリックで開くファイルのパスを入れてください。
「アイコン」
ボタンに貼り付けるアイコン画像です。
「概要画像」
作品のタイトルや簡単な遊び方などを記した概要イメージです。
「詳細テキスト」
著作権その他の情報をまとめたテキストファイルです。

　作品データの追加をしたいときは左下の「追加」ボタン (Alt+D) で追加できます。
　作品データの削除をしたいときは表から修正・削除したい項目を選んで、左下の「削除」ボタン (Alt+R) で削除できます。
　終わったら右下の "OK" または「適用」で保存できます。

　ファイルを指定するときはこんな感じで "Refer" ボタンが現れます。このボタンを押すと…

こんな感じでダイアログが出るので、これでファイルを選べば指定できます。

ログについて

　Digi Laun を展示モードで起動すると、 Log.csv というファイルに作品の起動ログを取ります。
　ログには、以下の列があります。

作品名ファイルステータス年月日時分秒エポックからの経過時間 [ms]
作品の名前起動したファイルへのパス
Started
アプリ起動成功
ExitSuccessful
アプリ正常終了
ExitFailed
アプリ異常終了
Killed
アプリ強制終了
Opened
ファイルを開いた
CannotOpen
起動不可
ログを取った時刻1970年1月1日午前0時0分0秒からの大体の経過時間
ログの時差からプレイ時間を計るのに使う（後述）

このログを使って各作品の起動回数、総プレイ時間、平均プレイ時間を計算できます。

計算例
  • 起動回数
    • ステータスが "Started" または "Opened" になっている回数を数える
  • 総プレイ時間
    • ステータスが "Started" になった時刻と "ExitSuccessful" になった時刻の差を求めて、その1回のプレイ時間を求める
      期間内に起動した時のプレイ時間を全部計算して足す
  • 平均プレイ時間
    • 合計プレイ時間÷起動回数

CleanUp.bat について

CleanUp.bat を実行すると、作品データに含まれる、不要と思われるファイルをすべて削除します。
一部の作品が動かなくなる場合があるので、削除するファイルを確認してください。
(打ち消し線が引いてあるのは、現在のバージョンではコメントアウトして削除されないようにしてあるファイルです。)

削除するファイル説明
BuildLog.htmVisual Studio のビルドログ
desktop.iniWindows がフォルダのカスタマイズ状態を保存するのに使う
folder.httWindows がフォルダ表示に特別な機能を追加するのに使う
Log.txtＤＸライブラリが出力するログ
Thumbs.dbWindows がサムネイル画像を保存するのに使う
*.bak何かのバックアップ
*.depDLL 製作時に、外部から呼び出し可能な関数を記述するらしい
*.embed.manifestよくわからないけど VC++ のビルド過程で出てくる
*.idbVC++ がビルドを高速化するときに使う
*.ilkVC++ が「インクリメンタルリンク」をするのに使う
*.intermediate.manifestよくわからないけど VC++ のビルド過程で出てくる
*.lnkショートカット
*.mak.makefile みたいなもの
*.objexe ファイルを作る過程で生まれたもの
*.pchVC++ でビルドを高速化するのに使う
*.pdbデバッグに使う
*.psdAdobe Photoshop で描いた画像
*.resexe に埋め込んで使う
*.sbrVC++ のビルド過程で出てくるらしい中間ファイル
*.suoVisual Studio ソリューションの比較的どうでもいい設定を保存している
*.tdsBorland C++ が吐き出すデバッグ情報
*.zip圧縮フォルダ
